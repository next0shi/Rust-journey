"What is Programming", "Building websites and products that serve a purpose on the internet"

"Why is it important", "It helps people get things done"

"What is the Internet?", "The Internet is a global network of interconnected computers that can communicate with each other using standardized protocols."

"What is the difference between the Internet and the Web (World Wide Web)?", "The Internet is the underlying infrastructure (network of computers). The Web is a service that runs on the Internet, consisting of websites and web pages accessed through browsers using HTTP/HTTPS."

"What is a web browser?", "A web browser is client software (like Chrome, Firefox, Edge) that requests web pages from servers, receives them, and displays them to the user."

"What is a web server?", "A web server is a computer (or software) that stores website files and sends them to clients (browsers) when requested."

"What does HTTP stand for and what is its main purpose?", "HyperText Transfer Protocol. It is the protocol used for communication between web browsers (clients) and web servers."

"What is the difference between HTTP and HTTPS?", "HTTPS is HTTP with encryption (using SSL/TLS). It secures data transferred between client and server, protecting sensitive information like passwords and credit cards."

"What is a URL?", "Uniform Resource Locator – the address used to locate a resource on the web (e.g., https://www.example.com/page)."

"What are the main parts of a URL?", "Protocol (https://), domain name (www.example.com), path (/page), and optionally query parameters (?id=123)."

"What is HTML?", "HyperText Markup Language, the standard language used to create and structure the content of web pages."

"What is the purpose of CSS?", "Cascading Style Sheets is used to control the visual presentation and layout of HTML elements (colors, fonts, spacing, etc.)."

"What is JavaScript used for in web development?", "JavaScript is a programming language used to add interactivity, dynamic behavior, and client-side logic to web pages."

"What is the client-server model in web development?", "In the client-server model, the client (browser) sends requests to the server, which processes the request and sends back a response (usually web pages or data)."

"What is a domain name?", "A human-readable address (like google.com) that is translated by DNS into an IP address so browsers can locate the correct web server."

"What does DNS stand for and what is its function?", "Domain Name System. It translates easy-to-remember domain names into the numerical IP addresses that computers use to communicate."

"What is an IP address?", "A unique numerical label assigned to every device connected to a network (e.g., 192.168.1.1) that identifies it for communication."

"What are web standards and why are they important?", "Web standards are rules and guidelines that ensure websites work consistently across different browsers and devices."

"The three core technologies of the World Wide Web.", "HTML (structure), CSS (presentation), and JavaScript (behavior)."

"What is the difference between a website and a web page?", "A web page is a single document. A website is a collection of related web pages under one domain name."

"What is a search engine and how does it differ from a web browser?", "A search engine helps users find websites. A web browser is the tool used to actually visit and display those websites."

"What happens when you type a URL in the browser and press Enter?", "1. DNS lookup converts domain to IP address. 2. Browser sends HTTP request to the server. 3. Server responds with HTML, CSS, JS files. 4. Browser renders the page."

"What are attributes in HTML? Give two examples.", "Attributes provide additional information about HTML elements. Examples: id="main" (unique identifier) and class="container" (for styling or grouping)."

"What is the purpose of the alt attribute in an <img> tag?", "The alt attribute provides alternative text that describes the image. It is displayed if the image fails to load and is important for accessibility (screen readers)."

"What is JavaScript and where can it run in web development?", "JavaScript is a programming language that adds interactivity to web pages. It can run on the client-side (in the browser) or server-side (with Node.js)."

"What is the difference between client-side and server-side scripting?", "Client-side scripting (JavaScript in browser) runs on the user’s device. Server-side scripting runs on the web server before sending content to the browser."

"What is an event in JavaScript? Give two examples.", "An event is an action that occurs in the browser, such as a user clicking a button (click) or typing in an input field (input)."

"What does the Document Object Model (DOM) represent?", "The DOM is a tree-like representation of the HTML document that JavaScript can read and modify dynamically."

"What is responsive web design?", "Responsive web design is an approach that makes web pages render well on a variety of devices and screen sizes (desktops, tablets, and mobile phones)."

"Why is it important to separate HTML, CSS, and JavaScript into different files in a real project?", "Separation of concerns makes the code easier to read, maintain, debug, and reuse. It also improves collaboration among developers."

"What is the main purpose of using domain names instead of raw IP addresses when accessing websites?", "Domain names provide human-readable addresses for web servers, solving the problems that IP addresses are hard to remember and may change over time."

"What does TLD stand for, and what role does it play in a domain name?.", "TLD stands for Top-Level Domain. It appears at the rightmost part of the domain and indicates the general purpose or category of the service (e.g., .com or .org for general use, .gov for government departments only, .edu for educational institutions)."

"Can you permanently “buy” and own a domain name, and why the system works this way.", "No, you cannot permanently own a domain name. You only pay for the right to use it for one or more years through a registrar. This ensures unused domains eventually become available again instead of being locked forever."

"Most basic, what is the Internet?", "A large network of computers that communicate with each other, allowing them to stay connected regardless of what happens to individual parts of the network."

"Why is a network switch used when connecting multiple computers in a local network, and what problem does it solve?", "A switch ensures that messages sent from one computer reach only the intended recipient. It solves the problem of needing a direct cable between every pair of computers (which would require 45 cables for 10 computers)."

"What is the role of a router when connecting multiple local networks?", "A router forwards messages (packets) between different networks by reading the recipient address, similar to how a post office routes mail. It prevents the need for an unmanageable tree of switches."

"What is an IP address, and why do we also use domain names?", "An IP address is a unique numerical identifier for a computer on the network (e.g., 192.0.2.172). Domain names are used because they are human-readable and much easier to remember than strings of numbers."

"What is the key difference between the Internet and the Web?", "The Internet is the underlying technical infrastructure (network of connected computers). The Web is a service built on top of the Internet that uses web servers to deliver webpages to browsers. Other services like email also run on the Internet."

"In the client-server model of the web, what are clients and what are servers?", "Clients are internet-connected devices and software (such as web browsers on computers or phones) that request resources. Servers are computers that store websites and send the requested files back to the client."

"What are the five main additional components (beyond client and server) needed for the web to function, and what is the role of each?", "1. Internet connection – the “road” that carries data. 2. TCP/IP – the protocols that ensure reliable data transmission. 3. DNS – the address book that converts domain names to IP addresses. 4. HTTP – the language clients and servers use to communicate. 5. Files – the HTML, CSS, JavaScript, and assets that make up the website."

"Why are data broken into packets when traveling over the Internet, and what information does each packet header contain?", "Packets allow efficient, reliable transmission: lost or corrupted packets can be requested individually, and they can take different routes to avoid congestion. Each header contains source and destination IP addresses, sequence number, total number of packets, and the protocols used."

"What does the HTTP status code 200 mean? Explain two other common status codes.", "200 means “OK” – the request succeeded and the resource was delivered. 301 = Moved Permanently (resource has a new permanent URL). 404 = Not Found (resource does not exist at that URL)."

"What is the difference between a web page, a website, and a web server?", "A web page is a single HTML document displayed in a browser. A website is a collection of linked web pages (and resources) that share a domain name. A web server is the physical computer that stores and delivers the website’s files to browsers."

"What is a search engine, and how does it differ from a web browser?", "A search engine is a web service (e.g., Google, Bing) that helps users find web pages. A web browser is the software that actually retrieves and displays those pages."

"Basic step-by-step process that occurs when you browse to a web page by typing a URL or clicking a link.", "1. The browser sends an HTTP request (usually GET) to the web server. 2. The server responds with the requested HTML file and any linked resources (images, CSS, JS). 3. The browser receives all resources, parses them, and renders the complete page for display."

"What is the difference between a website and a web service?", "A website primarily serves HTML pages for users to view. A web service responds to requests to perform functions or provide data (e.g., APIs, image resizing, weather data) and may or may not serve visible web pages."

"Why is it important for web developers to understand the distinction between a browser, a website, a web server, and a search engine?", "Clear terminology prevents confusion when troubleshooting (e.g., “the website is down” usually means the web server is down) and helps developers communicate effectively and debug issues correctly."
